## 🔍 核心实现原理

### 1. TSX 实时编译

使用 `@babel/standalone` 在浏览器端将 TSX 代码转换为可执行的 JavaScript。编译流程：用户输入 → 防抖 300ms → Babel 转换 → 生成 Blob URL → 注入 iframe。

```typescript
import Babel from '@babel/standalone';

const result = Babel.transform(code, {
    presets: ['react', 'typescript'],
    filename: 'App.tsx'
});
```

编译错误通过正则解析 Babel 的报错信息，提取行号和列号，在 Monaco 编辑器中以内联标记（`setModelMarkers`）的形式实时显示红色波浪线。

### 2. 预览渲染

将编译后的代码封装为 Blob URL，注入 iframe 实现隔离渲染。iframe 使用 `sandbox="allow-scripts"` 确保安全隔离，同时通过 `postMessage` 与父通信传递控制台日志和运行时错误。

```typescript
const blob = new Blob([compiledCode], { type: 'application/javascript' });
const url = URL.createObjectURL(blob);
```

CSS 文件的修改采用热更新策略：当检测到 `.css` 文件变化时，直接通过 `postMessage` 通知 iframe 替换 `<style>` 标签内容，无需重新编译整个应用。

### 3. TypeScript 智能提示

通过 `@typescript/ata`（Automatic Type Acquisition）自动获取第三方库类型定义。ATA 在后台 Worker 中运行，根据代码中的 `import` 语句自动下载对应的 `@types/*` 包，提供精准的代码补全和类型检查。

```typescript
import { createTypeAcquisitionWorker } from '@typescript/ata';

const ata = createTypeAcquisitionWorker({
    worker: new Worker(new URL('./ata.worker.ts', import.meta.url))
});
```

### 4. 样式隔离

采用 CSS Modules + Sass 方案，每个组件的样式通过 `.module.scss` 文件定义，编译后自动加上唯一的哈希类名，确保组件样式互不干扰：

```typescript
import styles from './Editor.module.scss';

<div className={styles.editorContainer}>...</div>
```

### 5. 防抖优化

使用 `lodash-es` 的防抖函数，避免频繁输入导致的重复编译。编译防抖 300ms，文件保存防抖 500ms，版本历史保存防抖 2s：

```typescript
import { debounce } from 'lodash-es';

const debouncedCompile = debounce(compileCode, 300);
```

### 6. WebSocket 实时协作

基于原生 `WebSocket` 实现多人实时编辑。后端使用 Next.js API Routes 处理 REST 请求，WebSocket 服务独立运行在 3001 端口。协议设计：

- **文件同步**：`FILE_UPDATE` 消息广播文件变更，基于 `uuid` 追踪修改来源，避免回环
- **光标位置**：`CURSOR_MOVE` 消息实时同步每个用户的光标位置和选区
- **权限管理**：Owner 可邀请 Editor/Viewer，不同角色有不同的编辑权限
- **房间管理**：创建 / 加入 / 离开房间，断线自动重连

```typescript
const ws = new WebSocket(`ws://localhost:3001/rooms/${roomId}`);
ws.onmessage = (event) => {
    const msg = JSON.parse(event.data);
    switch (msg.type) {
        case 'FILE_UPDATE': /* 同步文件变更 */ break;
        case 'CURSOR_MOVE': /* 更新光标位置 */ break;
        case 'USER_JOIN':   /* 新用户加入 */ break;
    }
};
```

### 7. IndexedDB 状态持久化

使用原生 IndexedDB 保存文件内容，浏览器关闭后重新打开仍可恢复上次编辑状态。存储结构：`react-playground` 数据库 → `files` 存储，key 为 `current`，value 包含完整的 `Files` 对象。同时使用 `localStorage` 保存用户偏好（主题、字体大小、语言等）。

```typescript
const db = await openDB('react-playground', 1);
await db.put('files', { id: 'current', files });
```

### 8. Code Splitting 与懒加载

通过 Vite 的 `manualChunks` 将大型依赖拆分为独立 chunk（React、Monaco Editor、Babel），同时使用 `React.lazy` + `Suspense` 对面板组件进行懒加载，首屏只加载核心编辑器和预览区，其他面板按需加载。

```typescript
const AIAssistant = React.lazy(() => import('./Components/AIAssistant'));
const CommandPalette = React.lazy(() => import('./Components/CommandPalette'));

<Suspense fallback={null}>
    <AIAssistant />
</Suspense>
```

### 9. Monaco Editor 主题系统

通过 `monaco.editor.defineTheme()` 动态注册自定义主题，每个主题定义 token 级别的语法高亮规则（comment、keyword、string、number、type、function 等），用户切换主题时实时调用 `setTheme()` 生效，选择持久化到 localStorage。

```typescript
monaco.editor.defineTheme('dracula', {
    base: 'vs-dark',
    inherit: true,
    rules: [
        { token: 'comment', foreground: '6272a4' },
        { token: 'keyword', foreground: 'ff79c6' },
    ],
    colors: {
        'editor.background': '#282a36',
    }
});
```

### 10. 响应式布局

采用 CSS 媒体查询 + React 状态双重方案实现移动端适配。SCSS 定义断点变量（`$breakpoint-tablet: 768px`、`$breakpoint-mobile: 480px`），React 通过 `window.innerWidth` 监听窗口宽度，动态调整布局方向（桌面端左右分栏 → 移动端上下分栏）和组件显隐（汉堡菜单、默认折叠面板）。

## 🧩 开发指南

### 添加新功能

1. 在 `src/Components/` 下创建新组件目录
2. 使用 `.module.scss` 编写样式，确保隔离
3. 通过 `PlaygroundContext` 共享全局状态
4. 遵循 TypeScript 类型规范，完善类型定义

### 修改默认模板

模板文件位于 `src/ReactPlayground/template/`：
